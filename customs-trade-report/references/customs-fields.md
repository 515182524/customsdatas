# 中国海关数据字段参考

## 常见字段

| 分析角色 | 常见列名 | 说明 |
|---|---|---|
| 时间 | 数据年月、年月、统计年月、月份 | 常见格式为 `YYYYMM` |
| 金额 | 美元、美元金额、金额（美元）、人民币 | 必须保留实际币种口径 |
| 商品 | 商品编码、商品名称、HS编码 | 商品编码应按文本读取，保留前导零 |
| 贸易伙伴 | 贸易伙伴名称、贸易伙伴、产销国 | 不能自动等同于原产国或目的国 |
| 原产地 | 原产国（地区）名称、原产国名称、原产地 | 通常用于进口分析 |
| 目的国 | 最终目的国（地区）名称、最终目的国名称、目的国 | 通常用于出口分析 |
| 国内区域 | 注册地名称、境内目的地、境内货源地 | 含义取决于查询口径 |
| 贸易方式 | 贸易方式名称、贸易方式 | 如一般贸易、进料加工贸易 |
| 数量 | 第一数量、第二数量 | 必须与对应计量单位一起使用 |
| 计量单位 | 第一计量单位、第二计量单位 | 不同单位不得直接相加 |
| 方向 | 进出口、进出口标志、贸易流向 | 缺失时无法单凭数据判断进口或出口 |

## 字段解释原则

1. 优先使用明确字段名，不从国家名称或商品名称猜测字段语义。
2. `贸易伙伴` 是海关查询中的伙伴维度，可能随查询口径代表产销国、原产地或最终目的国，但必须由源数据或用户确认。
3. `注册地名称` 通常是企业注册地，不等于生产地、消费地或口岸。
4. `境内货源地` 与 `境内目的地` 具有不同业务含义，不应统一改名为省份来源。
5. 商品编码可能包含前导零，读取时始终按文本处理。
6. 海关导出 CSV 常使用 GB18030 编码，并可能带有多余空列。
7. 第一数量对应第一计量单位，第二数量对应第二计量单位。两组数量是平行口径，不代表可直接换算。
8. 常见组合包括“台 + 千克”“个 + 千克”等，可分别形成数量单价与重量单价。

## 数据质量检查

- 金额或时间字段缺失
- 时间格式无法解析
- 金额、数量中的逗号、空白、负数或非数字
- 最新月份记录量或金额明显偏低
- 商品编码位数不一致
- 数量大于零但单位缺失
- 同一数量字段包含多个计量单位
- 数量为 0、空值或负数时，对应记录不得进入单价计算
- 单价计算所覆盖的金额占总金额比例过低
- 只有单月数据却要求趋势或同比分析
