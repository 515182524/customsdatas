# 海关贸易查询与分析

把用户的产品描述转换为经过多源搜索和可靠来源核验的 8 位 HS 编码，下载海关源数据，并生成可复核的 `report.md`。

## 默认口径

- 进出口方向：用户未说明时默认 **出口**
- 币制：用户未说明时默认 **美元**
- 时间：用户未说明时默认 **2026 年 1-12 月**
- 若官网尚未发布全年数据，自动使用官网当前可查询的最后月份，并按实际数据时间命名
- 固定勾选“分月展示”，固定使用完整贸易伙伴、贸易方式和收发货人注册地编码

## 1. 确认 HS 编码

用户已提供 8 位 HS 编码时，仍应通过官方商品参数验证商品名称是否匹配。

用户只提供产品描述时：

1. 提取商品核心名称、材质、用途、结构、加工状态等归类要素。
2. 使用浏览器搜索产品名称、英文名称、同义词及可能的 HS 编码。优先参考海关、税则、政府或专业报关来源；普通搜索结果只用于发现候选，不直接视为最终结论。
3. 使用海关官网商品参数检索候选名称或编码进行核验：

```bash
python3 .agents/skills/customs-trade-report/scripts/search_hs_candidates.py \
  "商品关键词" --year 2026 \
  --output "00- 草稿/海关数据报告/hs-candidates.json"
```

4. 比较多源候选、可靠来源说明和官方 8 位商品名称，直接选择最大可能性的编码。
5. 存在多个合理编码时不向用户补问；使用最常见产品形态和搜索结果中的主流含义作默认判断，并明确标注这是最大可能性推断，不宣称“绝对准确”。
6. 向用户简要说明最终 HS 编码、官方商品名称、搜索来源和选择依据，然后直接继续查询下载。

## 2. 查询、下载并分析

确认 HS 编码后运行端到端脚本：

```bash
python3 .agents/skills/customs-trade-report/scripts/run_customs_trade_report.py \
  --hs-code 84313900 \
  --flow 出口
```

可选参数：

- `--flow`：`进口`、`出口`、`进出口`；默认 `出口`
- `--currency`：`人民币`、`美元`；默认 `美元`
- `--year`：默认 `2026`
- `--start-month`、`--end-month`：默认 `1`、`12`
- `--output-root`：默认 `00- 草稿/海关数据报告`

网站可能出现拼图验证码。出现时由用户手动滑动；脚本检测验证成功后继续点击确定、等待数据、导出源表并生成报告。不得自动破解验证码。

若结果页长时间停在“数据加载中，请稍等”且没有生成总条数，脚本等待 45 秒后自动：

1. 点击“返回设置”
2. 保留原查询条件并重新点击“查询”
3. 停在验证码处等待用户手动验证
4. 验证通过后继续等待数据并下载

自动恢复只重试一次，避免循环重复查询。

### 浏览器会话管理

- 每个一次性浏览器会话用完后立即关闭，不要等所有任务结束后统一关闭。
- HS 编码官方核验完成并保存结果后，立即关闭该核验网页。
- 每次查询分段完成下载并保存下载记录后，立即关闭该查询网页，再开始下一分段。
- 脚本失败、超时或异常退出时也必须关闭当前一次性会话。
- 只有 `fill-only`、等待用户手动验证码或用户明确要求保留网页时，才暂时保持会话打开；完成后立即关闭。
- 不使用 `close-all` 关闭其他任务或用户正在使用的浏览器，只关闭当前脚本创建的命名会话。

### 最少验证码的分段导出

海关网站单次最多导出 10,000 条。查询较长时间范围时，优先减少用户处理验证码的次数：

1. 先按用户要求查询完整时间范围并导出一次。
2. 比较结果页总条数与实际导出行数；仅当总条数大于导出行数时，判定文件被截断。
3. 读取首份文件覆盖的最后一个月份，将该月份视为不完整月份并从首份文件中删除，只保留此前完整月份。
4. 第二次只查询“被删除的月份至原结束月份”，不要预先按季度或半年拆分。
5. 合并首份保留数据与补段数据；若补段仍被截断，按同样规则继续，直到覆盖完整时间范围。
6. 若单独一个月份仍超过 10,000 条，停止自动月份拆分并明确报告该限制。

例如：全年结果共 18,000 条，首次导出 10,000 条且包含 1-7 月，则删除首份中的 7 月，只保留 1-6 月；第二次仅查询 7-12 月。正常情况下用户只需处理两次验证码。

## 输出约定

每次查询输出到同一个结果目录：

```text
00- 草稿/海关数据报告/84313900_202601-202604_出口/
├── 84313900_202601-202604_出口.csv
├── report.md
├── result.json
└── charts/
    ├── monthly-trend.svg
    ├── partner-top.svg
    └── ...
```

源表命名规则：`{8位HS编码}_{实际起始年月}-{实际结束年月}_{进口|出口|进出口}.csv`。

`result.json` 记录源表、报告和结果目录的绝对地址。海关 CSV 通常使用 `GB18030` 编码。

## 分析规则

- 先读 [references/customs-fields.md](references/customs-fields.md)，确认字段语义。
- 需要解释指标或选择图表时，读 [references/analysis-methodology.md](references/analysis-methodology.md)。
- Markdown 报告中的每张图片前后、每个表格前必须保留一个空行，避免图片、说明和表格紧贴。
- `贸易伙伴` 不自动等同于 `原产国` 或 `最终目的国`。
- 金额列不是美元时，保留实际币种，不擅自换算。
- 月度加权单价是金额合计除以数量合计，不使用逐行单价简单平均。
- 第一数量、第二数量分别匹配各自计量单位，不互相换算。
- 相关性不得写成因果；最新月份不完整时必须提示。

## 完成检查

- HS 编码为 8 位数字，有浏览器搜索证据，并经过海关商品参数或其他可靠税则来源核验。
- 方向、币制和实际时间范围与用户要求一致；未说明时使用默认口径。
- 源表文件名使用实际数据年月，而不是尚未发布的请求月份。
- 总条数超过单次导出上限时，已按首份文件末月自动补段；合并结果没有遗漏或重复边界月份。
- 每个一次性浏览器会话均已在使用完成后立即关闭，没有遗留会话，也没有误关其他会话。
- 源表可用 `GB18030` 读取，金额总计与报告汇总一致。
- `report.md` 和所有本地 SVG 图片可打开。
- 最终返回源表路径、报告路径，并概括 3-6 条重要发现。
